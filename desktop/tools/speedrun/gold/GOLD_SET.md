# Gold set v1 — known-correct CFA Level I question/answer pairs

`gold_set_v1.jsonl` is the measurement instrument for the card checker
(`card_check.py`, challenge 7f): 50 hand-written question/answer pairs with
known-correct answers, plus 15 deliberately seeded DEFECT pairs that test the
checker's sensitivity. One JSON object per line.

## Schema

| field            | type   | required | notes                                                            |
| ---------------- | ------ | -------- | ---------------------------------------------------------------- |
| `id`             | string | yes      | `gold::<topic-short>::NN` or `defect::<type>::NN`, unique        |
| `topic`          | string | yes      | one of the 10 probe-harness topic slugs (see below)              |
| `answer_type`    | string | yes      | `qualitative` \| `quantitative`                                  |
| `question`       | string | yes      | a self-contained exam-style question                             |
| `correct_answer` | string | yes      | the known-correct answer (for defects: the SEEDED-DEFECT answer) |
| `rationale`      | string | yes      | 1-2 sentences with the fact or the full computation              |
| `source_note`    | string | yes      | names the standard concept; no unverifiable page numbers         |
| `defect_type`    | string | defects  | `wrong` \| `vague` \| `duplicate` (absent on the 50 gold pairs)  |
| `based_on`       | string | no       | for seeded defects derived from a specific gold record           |

Topic slugs (exactly the `TOPICS` tuple in `probe_harness.py`, 5 gold records
each): `ethics`, `quantitative_methods`, `economics`,
`financial_statement_analysis`, `corporate_issuers`, `equity_investments`,
`fixed_income`, `derivatives`, `alternative_investments`,
`portfolio_management`.

Mix: 28 qualitative / 22 quantitative (56% / 44%). Every quantitative
answer is arithmetically re-derivable from its own rationale;
`tests/test_card_check.py` recomputes all 22 with hard-coded arithmetic and
fails if any stated answer drifts.

## Seeded defects (15)

- 5 × `wrong` — factually wrong answers: a factor inversion
  (`defect::wrong::01`), an omitted income component (`::02`), an inverted
  relationship (`::03`), an off-by-context standard number (`::04`), and a
  plain arithmetic error deliberately phrased with **no near-identical gold
  twin** (`::05`) so that a stdlib checker has nothing to contradict it
  against — it is EXPECTED to be missed and exists to keep the reported
  sensitivity honest.
- 5 × `vague` — "What is finance?"-grade questions, non-answers ("It depends
  on many factors."), and trivial answer-leaked-in-question pairs.
- 5 × `duplicate` — near-copies of named gold records (`based_on`) with
  trivial rewording only.

Defect records carry `SEEDED DEFECT` markers in `rationale` and
`source_note` and must never be studied or emitted into any deck; the
checker treats only the 50 non-defect records as the trusted reference.

## Authorship and honesty note

Hand-written for this fork on 2026-07-05 by the card-check workstream (not
generated by the AIG pipeline). Correctness was author-verified against
standard, common-knowledge CFA Level I material — definitions, standard
numbers, and formulas that appear in any Level I text (Macaulay/modified
duration, TVM, FIFO/LIFO, CAPM, put-call parity, GIPS basics, the Standards
of Professional Conduct by number and name). No licensed CFA Institute text
was consulted or copied while authoring; no copyrighted exam question is
reproduced; `source_note` deliberately names concepts rather than page
numbers that could not be verified offline. The deliberate bias is toward
bulletproof basics over impressive obscurity: a charterholder reading these
should find them easy, not wrong.
